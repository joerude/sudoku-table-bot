# 🧩 Sudoku League Bot

Телеграм-бот, который ведёт учёт соревновательной игры в судоку
([usdoku.com](https://www.usdoku.com/)) между друзьями: очки, места, сезоны —
вместо ручного Google Sheets.

## Возможности

- **Регистрация игроков** — `/join`, привязка ника usdoku — `/setnick`
- **Создание игры** — `/newgame medium hardcore` → бот создаёт **реальную игру на
  usdoku** и присылает код + ссылку
- **🤖 Авто-учёт** — бот сам опрашивает usdoku и записывает результат, когда вы
  доиграли (порядок берётся из времени финиша)
- **Запись вручную** — `/result`, тап по игрокам в порядке финиша (fallback)
- **Очки** — настраиваемая таблица на сезон (дефолт `3 / 1 / 0`)
- **Сезоны** — до порога (дефолт 100 очков), затем авто-архив и старт нового
- **Удаление с подтверждением и возвратом** — soft delete, можно вернуть в один тап
- **Статистика** — `/status`, `/season`, `/me`, `/history`
- **Напоминания** — ловит незаписанные игры

## Быстрый старт (Docker)

1. Получи токен у [@BotFather](https://t.me/BotFather).
2. Скопируй конфиг и впиши токен:
   ```bash
   cp .env.example .env
   # отредактируй BOT_TOKEN в .env
   ```
3. Запусти:
   ```bash
   docker compose up -d --build
   ```
4. Добавь бота в групповой чат и напиши `/start`.

Данные лежат в `./data/sudoku.db` (примонтированный volume), переживают перезапуск.

## Добавить бота в чат

1. Создай бота у [@BotFather](https://t.me/BotFather) (`/newbot`) и вставь токен в `.env`.
2. **В группу:** открой группу → **Добавить участников** → найди бота по нику
   (например `@sudoku_table_bot`) → добавь.
3. В группе: `/start`, затем каждый игрок делает `/join` и
   `/setnick <ник_на_usdoku>` (ник должен совпадать с тем, что вводите при входе
   в игру на usdoku — иначе авто-учёт не свяжет результат с игроком).
4. Прав администратора боту **не нужно**. Приватный режим (бот видит только
   команды, а не всю переписку) включён у новых ботов по умолчанию — этого хватает.

При добавлении в группу бот сам пишет приветствие с быстрыми кнопками, а **меню
команд** (кнопка «/») он регистрирует автоматически при старте — ручной
`/setcommands` в @BotFather не нужен.

> ⚠️ В группе команды набирай **`/start`** (или жми из меню «/»), а не
> «@бот команда» — «@бот /start» Telegram шлёт как обычный текст, и это не команда.

Для теста один-на-один можно просто открыть бота в личке и написать `/start`.

## Локальный запуск (без Docker)

```bash
export BOT_TOKEN=...   # токен от BotFather
export DB_PATH=./data/sudoku.db
go run ./cmd/bot
```

## Тесты

```bash
go test ./...
```

## Команды

| Команда | Описание |
|---------|----------|
| `/join [имя]` | Зарегистрироваться |
| `/setnick <ник>` | Привязать ник usdoku (для авто-учёта) |
| `/players` | Список игроков |
| `/newgame <сложность> [режим]` | Создать игру на usdoku + ссылка |
| `/result` | Записать результат вручную (тап по порядку финиша) |
| `/status` | Таблица текущего сезона |
| `/season` | Инфо о сезоне |
| `/me` | Личная статистика |
| `/history` | Последние игры |
| `/settings` | Настройки: порог, таблица очков, напоминания (админ) |

## Команды разработки (just)

[`just`](https://github.com/casey/just) — таск-раннер. Установка:
`brew install just`, `cargo install just` или `apt install just`.
Полный список — `just` (или `just --list`).

| Команда | Что делает |
|---------|------------|
| `just run` | Локальный запуск (из `.env`, БД в `./data`) |
| `just check` | `build` + `vet` + `test` (перед коммитом) |
| `just test` | Тесты |
| `just fmt` | Форматирование кода |
| `just up` | Собрать образ и поднять в Docker |
| `just rebuild` | Чистая пересборка (`--no-cache`) + пересоздание |
| `just logs` | Живые логи контейнера |
| `just restart` | Перезапустить без пересборки |
| `just down` | Остановить и удалить контейнер |
